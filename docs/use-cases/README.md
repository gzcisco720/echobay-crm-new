# EchoBay CRM — 用户用例总览

本目录按角色拆分，用大白话描述每个人在系统里能做什么、怎么做、做完有什么结果。

---

## 系统中的三类用户

| 角色 | 文件 | 说明 |
|------|------|------|
| 管理员（Admin / Super Admin） | [uc-admin.md](./uc-admin.md) | 负责邀请商家、审核申请、管理品牌/门店/促销/首页产品 |
| 商家（Merchant） | [uc-merchant.md](./uc-merchant.md) | 受邀注册后，管理自己的品牌、门店、促销活动和文件 |
| 申请人（Applicant） | [uc-applicant.md](./uc-applicant.md) | 还没有账号，通过邀请链接注册并填写入驻申请 |

---

## 典型业务流程（全链路）

```
管理员发邀请
    ↓
申请人收到邮件，点链接注册账号
    ↓
申请人填写多标签申请表，上传 Logo，提交
    ↓
管理员在后台看到新申请，审核内容
    ↓
管理员批准 → 系统自动生成品牌档案，商家可以登录管理后台
管理员拒绝 → 申请关闭
管理员要求补充信息 → 商家看到提示，修改后重新提交
    ↓
品牌激活后，管理员/商家可以：
  - 管理门店（添加/编辑/删除）
  - 管理促销活动
  - 管理文件上传
  - 管理首页产品（仅管理员）
```
